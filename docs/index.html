g1_light_tracking

Dokumentacja repozytorium systemu percepcji, śledzenia i logiki misji dla robota intra-logistycznego. Strona została podzielona na zakładki, zawiera interaktywny diagram przepływu modułów, panel uruchamiania sekwencji oraz dokładny opis komunikacji ROS 2.

ROS 2 Python OpenCV YOLOv8n QR AprilTag CLI GUI Git hooks

Nawigacja

Przegląd projektu

Repozytorium dostarcza bazową architekturę dla robota, który potrafi wykrywać człowieka, karton, QR kod, AprilTagi oraz plamkę światła, a następnie powiązać dane percepcyjne z obiektem logistycznym reprezentowanym przez ParcelTrack.

Percepcja

YOLO, QR, AprilTag, plamka światła, kolor.

Tracking

Stabilizacja w czasie i utrzymanie tożsamości obiektów.

Logistyka

Wiązanie QR z kartonem i budowa ParcelTrack.

Misja

Maszyna stanów od wyszukiwania do odłożenia przesyłki.

Przepływ danych

Architektura modułów

perception_node

Detekcja 2D: YOLO, QR, AprilTag, plamka światła.

localization_node

Pozycja 3D, bbox, propagacja do dalszych etapów.

tracking_node

Tracking w czasie, w tym filtr Kalmana dla wybranych klas.

parcel_track_node

Kojarzenie QR do kartonu i publikacja ParcelTrack.

mission_node

Wybór celu i maszyna stanów wysokiego poziomu.

control_node

Przeliczanie celu na uproszczone komendy ruchu.

debug_node

Logowanie i diagnostyka.

standalone CLI/GUI

Uruchamianie bez topiców ROS 2.

Kalibracja kamery

Repozytorium zawiera osobny moduł poprawnej kalibracji kamery oparty o planszę szachownicy. Moduł zbiera próbki narożników, estymuje intrinsics przez cv2.calibrateCamera, publikuje CameraInfo oraz zapisuje wynik do pliku YAML.

Pliki

  • g1_light_tracking/nodes/camera_calibration_node.py
  • scripts/camera_calibration_node
  • config/camera_calibration.yaml

Wejścia / wyjścia

  • Wejście: /camera/image_raw
  • Wyjście: /camera/camera_info_calibrated
  • Podgląd: /debug/calibration_preview
  • Plik wynikowy: calibration/camera_intrinsics.yaml
ros2 run g1_light_tracking camera_calibration_node --ros-args --params-file config/camera_calibration.yaml

W GUI standalone dostępne jest dodatkowe okno Top-Down, które pokazuje lokalny podgląd odometrii w rzucie z góry.

Automatyczna zmiana wersji przy commicie

Repozytorium zostało rozszerzone o prosty mechanizm automatycznego zwiększania wersji patch przy każdym commicie.

  • Plik VERSION przechowuje bieżącą wersję.
  • Skrypt scripts/version_bump.py zwiększa patch i aktualizuje:
    • VERSION
    • setup.py
    • package.xml
  • Skrypt scripts/install_git_hooks.sh instaluje hook pre-commit.
bash scripts/install_git_hooks.sh

Po instalacji hooka każdy commit automatycznie podniesie numer wersji patch i doda zmienione pliki do commita.
