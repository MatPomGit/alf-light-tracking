g1_light_tracking

Dokumentacja repozytorium systemu percepcji, śledzenia i logiki misji dla robota intra-logistycznego. Dokumentacja została przeniesiona do katalogu docs/ i rozszerzona o dodatkowe grafy zależności, opisy architektury oraz rozdział o global motion compensation.

ROS 2 Python OpenCV YOLOv8n Global Top-Down Optical Flow / Homography

Nawigacja

Przegląd projektu

Repozytorium dostarcza architekturę dla robota, który wykrywa człowieka, karton, QR, AprilTagi i plamkę światła, a następnie wiąże dane percepcyjne z obiektem logistycznym ParcelTrack.

Percepcja

YOLO, QR, AprilTag, plamka światła, kolor.

Tracking

Kalman + utwardzenie przeciw false positives + kompensacja ruchu globalnego.

Logistyka

Wiązanie QR z kartonem i budowa ParcelTrack.

Top-down

Widok globalny z siatką metryczną, legendą oraz nakładaniem celów.

Architektura modułów

perception_node

Detekcja 2D: YOLO, QR, AprilTag, plamka światła.

localization_node

Pozycja 3D, bbox, propagacja do dalszych etapów.

tracking_node

Tracking w czasie, filtr Kalmana, gating bbox oraz GMC.

parcel_track_node

Kojarzenie QR do kartonu i publikacja ParcelTrack.

mission_node

Maszyna stanów wysokiego poziomu.

topdown_odom_viewer_node

Widok globalny z /odom, TF, siatką i nakładkami.

Global motion compensation

Tracking wykorzystuje teraz kompensację ruchu globalnego obrazu opartą o:

  • detekcję punktów charakterystycznych,
  • optical flow Lucas-Kanade,
  • homografię estymowaną metodą RANSAC,
  • fallback do medianowego przesunięcia, gdy homografia nie jest wystarczająco pewna.
/debug/calibration_preview (Image)
  -> tracking_node
    -> GlobalMotionCompensator
      -> /tracking/global_motion_debug
      -> compensated bbox / compensated center_u,v

Dzięki temu śledzenie jest znacznie mniej wrażliwe na kołysanie głowy robota podczas chodzenia.

Kamera głębi: auto-detekcja, mapowanie i nawigacja

Oprogramowanie automatycznie sprawdza, czy napływają aktualne klatki z kamery głębi. Jeżeli tak, uruchamia tor depth-aware:

  • LocalizationNode używa depth ROI w obrębie detekcji do lepszej estymacji XYZ.
  • DepthMapperNode buduje lokalną siatkę zajętości OccupancyGrid.
  • ControlNode wykorzystuje DepthNavHint do omijania przeszkód i modulacji prędkości.
/camera/depth/image_raw
  -> localization_node (depth-assisted XYZ)
  -> depth_mapper_node
       -> /mapping/local_depth_grid
       -> /navigation/depth_hint
            -> control_node

Gdy depth nie jest dostępny, system automatycznie wraca do trybu RGB-only i zachowuje dotychczasowe działanie.

Wizyjny SLAM / Visual Odometry

Repozytorium zostało rozszerzone o visual_slam_node, który poprawia nawigację przez estymację ruchu robota na podstawie obrazu.

  • Monocular VO: ORB + dopasowanie cech + essential matrix + recoverPose.
  • RGB-D VO: ORB + głębia + solvePnPRansac, gdy depth jest dostępny.
  • Publikowane topiki:
    • /visual_slam/odom
    • /visual_slam/path
    • /visual_slam/debug_image
